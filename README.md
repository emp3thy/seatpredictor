# seatpredictor

UK Westminster seat predictor with tactical-consolidation modelling.

See `docs/superpowers/specs/2026-04-25-seat-predictor-design.md` for the full design.

## Setup

```bash
uv venv
uv pip install -e ".[dev]"
```

## Run tests

```bash
uv run pytest
```

## Generate a snapshot

```bash
uv run seatpredict-data snapshot
```

## Quick verification

After install, run:

```bash
uv run seatpredict-data fetch
uv run seatpredict-data snapshot
uv run python scripts/smoke_verify.py
```

The smoke verification asserts: 650 constituencies parsed, shares sum to ~100% per seat, ≥30 polls extracted, all four by-elections seeded, transfer matrix non-empty. If any check fails, the error message indicates which parser to inspect.

## Snapshots

Snapshots land in `data/snapshots/<as-of>__v<schema_version>__<content_hash>.sqlite`. Each is a self-contained SQLite file with seven tables: `manifest`, `polls`, `results_2024`, `byelections_events`, `byelections_results`, `transfer_weights`, `transfer_weights_provenance`. Use `data_engine.sqlite_io.open_snapshot_db` and `read_manifest` / `read_dataframe` to consume them programmatically.

Bumping `data_engine.snapshot.SCHEMA_VERSION` invalidates all old snapshot caches. Bump it whenever a parser change affects output (the input hash captures inputs, not parser code).

## Run a prediction

After producing a snapshot, run the prediction engine:

```bash
.venv/Scripts/seatpredict-predict.exe list-strategies
.venv/Scripts/seatpredict-predict.exe run \
    --snapshot data/snapshots/<snapshot-file>.sqlite \
    --strategy reform_threat_consolidation \
    --out-dir data/predictions \
    --multiplier 1.0 --clarity-threshold 5.0 --label baseline
```

Add `--reform-polling-correction-pp <pp>` to apply a pollster-bias correction to Reform's projected national share before per-seat projection. Recommended value is in `data/derived/reform_polling_bias.json` (produced by `notebooks/05_reform_polling_bias.ipynb`); positive value = pollsters under-state Reform.

(PowerShell: `.venv\Scripts\seatpredict-predict.exe ...`. POSIX: `.venv/bin/seatpredict-predict ...`.)

**Why the direct binary path?** `uv run seatpredict-predict` triggers a package rebuild that reverts the editable install to PEP 660 finder mode, which fails to expose `prediction_engine`. If you see `ModuleNotFoundError`, run:

```bash
uv pip install --config-settings editable_mode=compat -e ".[dev]"
```

Outputs a single SQLite file at `data/predictions/<snapshot_hash>__<strategy>__<config_hash>__<label>.sqlite` containing four tables: `seats`, `national`, `config`, `notes_index`. Per-seat schema is documented at the top of `prediction_engine/sqlite_io.py`.

Sweep over multipliers:

```bash
.venv/Scripts/seatpredict-predict.exe sweep \
    --snapshot data/snapshots/<snapshot-file>.sqlite \
    --strategy reform_threat_consolidation \
    --out-dir data/predictions \
    --multiplier 0.5,0.75,1.0,1.25,1.5
```

Diff two runs:

```bash
.venv/Scripts/seatpredict-predict.exe diff data/predictions/<run-a>.sqlite data/predictions/<run-b>.sqlite
```

## Analyze predictions

```bash
.venv/Scripts/seatpredict-analyze.exe drilldown --run data/predictions/<run>.sqlite --seat E14000123 --explain
.venv/Scripts/seatpredict-analyze.exe flips     --runs data/predictions/<run-a>.sqlite data/predictions/<run-b>.sqlite
```

The `drilldown --explain` form prints a human-readable seat report (raw -> predicted shares, consolidator, clarity, matrix provenance, notes). Without `--explain`, it emits indented JSON for piping into other tools.

## Notebooks (first time)

Five analysis notebooks live in `notebooks/`, generated by `scripts/build_notebooks.py`:

| file | what it does |
|---|---|
| `01_polling_trends.ipynb` | per-party 7-day rolling poll average; sanity-check the data engine |
| `02_constituency_drilldown.ipynb` | one seat: raw shares -> consolidator -> flows -> predicted shares |
| `03_strategy_comparison.ipynb` | uniform_swing vs reform_threat_consolidation; flips and bar charts |
| `04_scenario_sweep.ipynb` | sweep `multiplier`; plot per-party seats as a function of multiplier |
| `05_reform_polling_bias.ipynb` | compare pre-event Reform polling against actual results (by-elections + local-election PNS); writes `data/derived/reform_polling_bias.json` with the recommended `--reform-polling-correction-pp` value |

To run them, register the project venv as a Jupyter kernel once:

```bash
uv pip install --config-settings editable_mode=compat -e ".[dev]"
uv run python -m ipykernel install --user --name python3 --display-name "Python 3 (seatpredictor)"
```

Then open them in either VS Code or JupyterLab:

**VS Code's notebook editor.** Open the `.ipynb` file in VS Code, click "Select Kernel" in the top-right, pick the `python3` kernel just registered. Run cells with Shift+Enter.

**JupyterLab in a browser.** From the repo root:

```bash
uv run jupyter lab
```

A browser tab opens at `http://localhost:8888`. Open `notebooks/01_polling_trends.ipynb` and run cells with Shift+Enter.

**To re-execute and refresh outputs from the command line:**

```bash
# Run from the repo root — relative paths in cells reference data/snapshots and data/predictions.
uv run jupyter nbconvert --to notebook --execute --inplace notebooks/01_polling_trends.ipynb
```

If the snapshot/prediction files don't exist yet, build them first via `seatpredict-data snapshot` and `seatpredict-predict run` (see sections above).

> **Note for first-time Jupyter users:** A notebook is a sequence of cells; each runs Python (or markdown). You run them top-to-bottom. The output (charts, tables) appears under each cell. Restarting the kernel clears all variables — use "Run All Cells" after a restart to re-execute everything.

## Regenerating the notebooks

The notebooks are committed to git, but their content is derived from `scripts/build_notebooks.py`. To edit a cell, modify the appropriate `_NB_*` constant in that script and rerun:

```bash
uv run python scripts/build_notebooks.py
```

Then strip outputs before committing so the diffs stay clean:

```bash
uv run jupyter nbconvert --clear-output --inplace notebooks/*.ipynb
```
